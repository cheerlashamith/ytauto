1. WHAT THIS PROJECT IS
=======================

This project is a local-first YouTube/course video automation system built around MoneyPrinterTurbo.

The goal is:

    Topic / URL / Autonomous idea
        -> AI script and scene plan
        -> generated visuals
        -> voiceover
        -> subtitles
        -> background music
        -> final MP4 video

The system uses:

    MoneyPrinterTurbo
        Main production/rendering engine.

    Ollama / Gemini / Groq / fallback
        Scene/script planning providers.

    ComfyUI
        AI visual background/story image generator.

    Python/Pillow overlay
        Clean readable text/diagram overlay for ComfyUI images.

    Manim
        Serious 3Blue1Brown-style technical course animation renderer.

    FFmpeg
        Clip conversion and final video processing.

3. WHAT FEEDBACK CHANGED THE ARCHITECTURE
=========================================

Mentor/sir feedback:

    Current educational videos look cartoon-like.
    They are more suitable for kids stories or storytelling.
    Serious course mode should look more like 3Blue1Brown animations.

Correct interpretation:

    The ComfyUI visual style is not wrong.
    It is just the wrong renderer for serious course mode.

Therefore:

    Keep ComfyUI for Story Mode.
    Add Manim for Course Mode.

This means we do NOT delete the old architecture.
We add a new visual renderer.


4. FINAL MODE-BASED ARCHITECTURE
================================

The platform should have these modes:

    1. Autonomous Mode
    2. YouTube URL Extraction Mode
    3. Story / Cartoon Mode
    4. Serious Course Mode
    5. Hybrid Mode

High-level structure:

    Input Sources
    ├── Manual Topic

    └── YouTube URL

            ↓

    Content Intelligence Layer
    ├── Research / transcript extraction
    ├── Script generation
    ├── Scene planning
    └── Visual style decision

            ↓

    Visual Router
    ├── Pexels stock footage
    ├── ComfyUI Story Mode
    ├── Manim Course Mode
    └── Hybrid Mode

            ↓

            ↓

5. WHAT EACH MODE USES
======================

5.1 Autonomous Mode
-------------------

Purpose:

    AI chooses topic and makes a video automatically.

Visual selection:

    Motivation/business/trend topic:
        Pexels or ComfyUI

    Technical/course topic:
        Manim

Example:

    Topic: Why discipline beats motivation
        -> Pexels/ComfyUI

    Topic: How binary search works
        -> Manim


5.2 YouTube URL Extraction Mode
-------------------------------

Purpose:

    User gives YouTube URL.
    System extracts transcript and creates a new explanation video.

Visual selection:

    Story/general URL:
        ComfyUI or Pexels

    Technical/course URL:
        Manim

Example:

    URL about recursion:
        -> transcript extraction
        -> AI rewrites lesson
        -> Manim call-stack animation
        -> MoneyPrinterTurbo final render


5.3 Story / Cartoon Mode
------------------------

Purpose:

    Kids stories, moral stories, cartoon explainers, simple narrative videos.

Renderer:

    ComfyUI

Looks like:

    cartoon teacher
    illustrated background
    story scenes
    character style
    once-upon-a-time narration

No need for strict number/diagram overlays.


5.4 Serious Course Mode
-----------------------

Purpose:

    Real educational course videos.

Renderer:

    Manim

Looks like:

    3Blue1Brown-style animations
    exact text
    exact numbers
    clean diagrams
    smooth transitions
    arrays / stacks / queues / graphs / formulas

Best for:

    algorithms
    data structures
    programming
    math
    operating systems
    DBMS
    networking
    AI/ML


5.5 Hybrid Mode
---------------

Purpose:

    Premium videos that need both art and exact diagrams.

Renderer:

    ComfyUI background + Manim diagram/overlay

Example:

    AI concept video with cinematic intro and exact neural network diagram.

9. STORY / CARTOON MODE USING COMFYUI
=====================================

Use when:

    kids story
    moral story
    cartoon explainer
    mythology story
    simple motivational video
    general visual explainer

How visuals are generated:

    ComfyUI generates AI backgrounds or character scenes.

Why no strict overlay needed:

    Story mode does not require exact arrays, formulas, pointers, or numbers.

MoneyPrinterTurbo role:

    Adds voice, subtitles, music, and final render.

===================================

    Linked List
    Binary Search
    Recursion
    Tree Traversal
    Graph BFS/DFS
    OS Scheduling
    DBMS Indexing
    Machine Learning concepts

Manim creates:

    exact numbers
    exact arrows
    exact arrays
    exact graph/tree structures
    smooth transitions

This is closest to 3Blue1Brown-style course animation.


11. MONEYPRINTERTURBO ROLE
==========================

MoneyPrinterTurbo remains the final production engine.

It should handle:

    TTS voice
    subtitles
    background music
    clip combination
    final MP4 output

It should NOT be replaced by Manim.

Correct responsibility split:

    Manim:
        course visual clips

    ComfyUI:
        story/illustration visuals

    MoneyPrinterTurbo:
        final video production


12. PROVIDERS EXPLANATION
=========================

12.1 Fallback
-------------

Status:

    Working

Cost:

    Free forever

Quality:

    Generic

Problem:

    Not enough topic-specific detail.

Use for:

    emergency demo
    provider failure


12.2 Ollama
-----------

Status:

    API running

Confirmed command:

    curl http://127.0.0.1:11434/api/tags

Problem:

    qwen3:14b can timeout.

Why:

    large model, long JSON prompt, slow local generation.

Fix ideas:

    use qwen3:8b
    reduce prompt length
    reduce num_predict
    use Groq/Gemini for scene JSON


12.3 Gemini
-----------

Status:

    Failed due to 429 Too Many Requests.

Meaning:

    Rate limit / quota issue.

Security:

    If keys were shared in logs/chat, revoke them.


12.4 Groq
---------

Status:

    Recommended next provider, but not implemented in current bridge yet.

Why useful:

    fast JSON generation
    good for scene planning
    less local timeout risk

Need:

    Groq API key
    code change in generate_scenes.py


13. WHY FALLBACK LOOKED BAD
===========================

Fallback creates generic scenes:

    Introduction
    Core Idea
    How It Works
    Example
    Key Parts
    Common Mistakes
    Real World Use
    Summary

For Stack, better scenes are:

    What is a Stack?
    LIFO Rule
    Push Operation
    Pop Operation
    Peek Operation
    Stack Example
    Real Uses
    Summary

So fallback is safe but not high quality.


14. WHY COMFYUI TEXT GETS GLITCHY
=================================

AI image models are bad at exact text.

They often create:

    broken letters
    fake labels
    distorted numbers
    wrong words

Solution:

    Do not ask ComfyUI to write text.
    Use Python/Pillow or Manim for exact text.

For course mode:

    Manim is better than ComfyUI overlays.

16. HOW FINAL VIDEO IS RENDERED
===============================

MoneyPrinterTurbo settings:

    Video Source: Local file
    Upload: generated clips
    Video Script: narration text
    Video Keywords: topic keywords
    Transition: None
    Aspect: 9:16 or 16:9
    TTS: Azure TTS V1
    BGM Volume: 0.05 to 0.1
    Subtitles: enabled

Output path:

    C:\MoneyPrinterTurbo-Portable-Windows-1.3.0\MoneyPrinterTurbo\storage\tasks\TASK_ID\final-1.mp4


17. HOW TO GENERATE A CUSTOM TOPIC RIGHT NOW
============================================

Safe current quality workflow:

    1. Create or edit scenes_auto.json manually.
    2. Make sure the script, titles, bullets, prompts are topic-specific.
    3. Run generate_materials.py.
    4. Upload clips into MoneyPrinterTurbo.

Commands:

    cd /d C:\MPT_ComfyUI_Full_Setup_Kit\MPT_ComfyUI_Full_Setup_Kit\01_auto_course_bridge
    notepad scenes_auto.json
    python generate_materials.py --scenes scenes_auto.json

For Manim course mode:

    cd /d C:\MPT_Manim_Course_Mode_Kit\manim_course_bridge
    run_stack_demo.bat

Then upload Manim MP4 to MoneyPrinterTurbo.


18. HOW TO USE MONEYPRINTERTURBO AFTER CLIP GENERATION
======================================================

Open:

    http://127.0.0.1:8501

Fill:

    Video Subject:
        your topic

    Video Script:
        generated moneyprinter_video_script.txt or manual script

    Video Keywords:
        generated moneyprinter_keywords.txt

    Video Source:
        Local file

    Upload Local Files:
        generated clips

    Transition:
        None

    Maximum Clip Duration:
        match generated clip duration

    TTS:
        Azure TTS V1

    BGM Volume:
        0.05 or 0.1

Then click Generate.
